# Implementation Plan - Sashakt Residential Autism Programme Website

## Goal
Build a high-conversion, empathetic, and professional website for a 1-Month Residential Autism Intervention Programme led by Rajini.

## Structure

### 1. File System
- `index.html`: The main landing page containing all key sections.
- `blog.html`: A dedicated page for resources and articles.
- `css/style.css`: Global styles, responsive design, and color variables.
- `js/script.js`: Interactive elements (mobile menu, smooth scroll, etc.).
- `assets/images/`: Directory for images (will use placeholders initially).

### 2. Design System
- **Color Palette**: 
  - Primary: Calming Blue (Trust, Professionalism) - e.g., `#2C7A7B` (Teal-ish) or `#3182CE`
  - Secondary: Soft Green (Growth, Calm) - e.g., `#68D391` or `#C6F6D5`
  - Background: Soft White/Off-White for high readability - e.g., `#F7FAFC`
  - Text: Dark Grey for contrast/accessibility - e.g., `#2D3748`
- **Typography**: Clean sans-serif fonts (Inter or Roboto) for high readability.

### 3. Page Content (`index.html`)
- **Hero Section**: 
  - Image: Rajini w/ child (Placeholder).
  - Headline: "From Dependence to Independence..."
  - CTA: "Apply for the Next Cohort".
- **Problem & Solution**:
  - Empathy regarding parental struggle.
  - "Behaviour Correction" & "Independent Human Being".
  - Methodology Icons/Text.
- **Program Structure**: 
  - 30 Days Residential.
  - Parent Partnership focus.
- **Pricing**: 
  - Comparison Cards (Package A vs B).
- **Blog Preview**: 
  - Links to articles.
- **Footer**: 
  - Contact, Map, Socials.

### 4. Blog Page (`blog.html`)
- List of articles with summaries and "Read More" links.
- Sidebar with categories.

## Step-by-Step Execution
1.  **Setup**: Create files and basic HTML structure.
2.  **Styles**: Implement the "Clean, Calming" CSS variables and utility classes.
3.  **Hero & Info**: Build the Hero, Problem/Solution, and Program sections.
4.  **Pricing & Blog**: Build the Pricing table and Blog preview.
5.  **Blog Page**: Create the dedicated blog page.
6.  **Review**: Ensure responsiveness and tone check.
